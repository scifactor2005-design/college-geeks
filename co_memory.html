Memory Hierarchy

Balancing speed, cost, and capacity

The Memory Pyramid

Computer memory is organized in a hierarchy from fastest/smallest to slowest/largest:

Level Type Size Access Time Cost/GB
L0 Registers ~1 KB ~0.25 ns Highest
L1 L1 Cache ~64 KB ~1 ns Very High
L2 L2 Cache ~512 KB ~4 ns High
L3 L3 Cache ~8 MB ~10 ns Medium
L4 Main Memory (RAM) 8-64 GB ~100 ns Low
L5 SSD 256 GB - 2 TB ~50 μs Very Low
L6 HDD 1-10 TB ~5 ms Lowest

Cache Memory

Small, fast memory between CPU and main memory. Exploits locality principles:

Temporal Locality

Recently accessed data is likely to be accessed again soon.

Spatial Locality

Data near recently accessed data is likely to be accessed soon.

Cache Organization:

  • Direct Mapped: Each memory block maps to exactly one cache line
  • Fully Associative: Any memory block can go to any cache line
  • Set Associative: Compromise between the two

Cache Performance:

Hit Rate: Percentage of memory accesses found in cache

Miss Rate: 1 - Hit Rate

Average Access Time = Hit Time + (Miss Rate × Miss Penalty)

Virtual Memory

Technique that allows execution of processes that may not be completely in physical memory. Provides illusion of large address space.

Benefits:

  • Allows programs larger than physical memory
  • Provides memory protection between processes
  • Enables efficient sharing of memory
  • Simplifies program loading

Paging:

Memory divided into fixed-size blocks called pages (typically 4KB).

  • Page Table: Maps virtual addresses to physical addresses
  • Page Fault: Occurs when requested page not in RAM
  • TLB (Translation Lookaside Buffer): Cache for page table entries

Memory Access Patterns

Sequential Access:

Accessing memory locations in order. Best cache performance.

Random Access:

Accessing memory locations in unpredictable order. Poor cache performance.

Optimization Techniques:

  • Loop Tiling: Breaking loops into smaller chunks
  • Data Alignment: Aligning data to cache line boundaries
  • Prefetching: Loading data before it's needed
  • Blocking: Organizing data to fit in cache

Applications

  • Database Systems: Buffer pool management
  • Operating Systems: Page replacement algorithms
  • Gaming: Texture and asset streaming
  • Scientific Computing: Optimizing matrix operations
  • Web Browsers: Caching web resources
