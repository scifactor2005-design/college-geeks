Instruction Set Architecture

The interface between hardware and software

What is ISA?

The Instruction Set Architecture defines the set of instructions a processor can execute. It's the contract between hardware and software, specifying how programs communicate with the CPU.

Components of ISA:

  • Instructions: Operations the CPU can perform
  • Data Types: Supported data formats (int, float, etc.)
  • Registers: Fast storage locations in CPU
  • Addressing Modes: Ways to specify operand locations
  • Memory Architecture: How memory is organized and accessed

Instruction Format

Instructions typically consist of:

  • Opcode: Operation to perform (ADD, SUB, LOAD, etc.)
  • Operands: Data or addresses to operate on
Example: ADD R1, R2, R3
- Opcode: ADD
- Destination: R1
- Source1: R2
- Source2: R3
Meaning: R1 = R2 + R3

Instruction Types:

  • Data Transfer: LOAD, STORE, MOVE
  • Arithmetic: ADD, SUB, MUL, DIV
  • Logical: AND, OR, NOT, XOR
  • Control Flow: JUMP, BRANCH, CALL, RETURN
  • Comparison: CMP, TEST

Addressing Modes

Different ways to specify where operands are located:

1. Immediate Addressing

Operand is part of the instruction itself.

MOV R1, #5    ; R1 = 5

2. Direct Addressing

Address of operand is given in instruction.

LOAD R1, 1000  ; R1 = Memory[1000]

3. Indirect Addressing

Instruction contains address of address.

LOAD R1, (R2)  ; R1 = Memory[R2]

4. Register Addressing

Operand is in a register.

ADD R1, R2, R3  ; R1 = R2 + R3

5. Indexed Addressing

Effective address = Base + Index

LOAD R1, 100(R2)  ; R1 = Memory[100 + R2]

RISC vs CISC

RISC (Reduced Instruction Set Computer)

  • Small, simple instruction set
  • Fixed instruction length
  • Load/Store architecture
  • More registers
  • Examples: ARM, MIPS, RISC-V

CISC (Complex Instruction Set Computer)

  • Large, complex instruction set
  • Variable instruction length
  • Memory-to-memory operations
  • Fewer registers
  • Examples: x86, x86-64

Example: Simple Assembly Program

; Calculate sum of array elements
    MOV R1, #0        ; R1 = sum = 0
    MOV R2, #1000     ; R2 = array base address
    MOV R3, #10       ; R3 = count = 10

LOOP:
    LOAD R4, (R2)     ; R4 = array[i]
    ADD R1, R1, R4    ; sum += array[i]
    ADD R2, R2, #4    ; Move to next element
    SUB R3, R3, #1    ; count--
    CMP R3, #0        ; Compare count with 0
    BNE LOOP          ; Branch if not equal

    STORE R1, 2000    ; Store result at address 2000
    HALT              ; Stop execution

Applications

  • Compiler Design: Translating high-level code to assembly
  • Operating Systems: Low-level system programming
  • Embedded Systems: Direct hardware control
  • Performance Optimization: Understanding CPU capabilities
  • Virtualization: Emulating different architectures
