Syntax Analysis

Parsing and validating code structure

Introduction

Syntax analysis (parsing) is the second phase of compilation. It takes tokens from lexical analyzer and builds a parse tree representing the grammatical structure of the program.

Context-Free Grammars (CFG)

CFGs define the syntax of programming languages using production rules.

E → E + T | T
T → T * F | F
F → (E) | id | num

Example derivation for "id + id * id":

E → E + T
  → T + T
  → F + T
  → id + T
  → id + T * F
  → id + F * F
  → id + id * F
  → id + id * id

Parsing Techniques

Top-Down Parsing

Starts from root and constructs tree downward. Examples: Recursive Descent, LL parsing.

Bottom-Up Parsing

Starts from leaves and builds up to root. Examples: LR, SLR, LALR parsing.

Parse Trees vs Syntax Trees

Parse Tree: Shows complete derivation with all grammar symbols

Abstract Syntax Tree (AST): Simplified tree showing essential structure, used in later compiler phases

Applications

  • Compiler construction
  • IDE syntax checking
  • Code formatters
  • Language processors
