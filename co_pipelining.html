Pipelining

Increasing instruction throughput through parallelism

What is Pipelining?

Pipelining is a technique where multiple instructions are overlapped in execution. Like an assembly line, different stages of different instructions execute simultaneously.

Classic 5-Stage Pipeline:

  1. IF (Instruction Fetch): Fetch instruction from memory
  2. ID (Instruction Decode): Decode instruction and read registers
  3. EX (Execute): Perform ALU operation
  4. MEM (Memory Access): Read/write data memory
  5. WB (Write Back): Write result to register

Pipeline Execution

Time →
Cycle: 1    2    3    4    5    6    7    8
I1:    IF   ID   EX   MEM  WB
I2:         IF   ID   EX   MEM  WB
I3:              IF   ID   EX   MEM  WB
I4:                   IF   ID   EX   MEM  WB
I5:                        IF   ID   EX   MEM

Without Pipeline: 5 instructions × 5 cycles = 25 cycles

With Pipeline: 5 + 4 = 9 cycles (ideal case)

Speedup ≈ Number of Stages (in ideal conditions)

Pipeline Hazards

1. Structural Hazards

Hardware cannot support all combinations of instructions simultaneously.

Solution: Add more hardware resources or stall the pipeline.

2. Data Hazards

Instruction depends on result of previous instruction still in pipeline.

ADD R1, R2, R3   ; R1 = R2 + R3
SUB R4, R1, R5   ; R4 = R1 - R5 (needs R1 from ADD)

Solutions:

  • Forwarding/Bypassing: Pass result directly to next instruction
  • Stalling: Insert bubbles (NOPs) in pipeline
  • Compiler Scheduling: Reorder instructions

3. Control Hazards (Branch Hazards)

Decision about control flow made while other instructions in pipeline.

Solutions:

  • Branch Prediction: Guess branch outcome
  • Delayed Branch: Execute instruction after branch regardless
  • Branch Target Buffer: Cache of branch targets

Performance Metrics

CPI (Cycles Per Instruction)

Average number of clock cycles per instruction.

Ideal Pipeline CPI = 1

Actual CPI = 1 + Stall Cycles per Instruction

Speedup

Speedup = Time without pipeline / Time with pipeline

Theoretical Max Speedup = Number of Pipeline Stages

Advanced Pipelining

Superpipelining

Divide pipeline stages into smaller stages for higher clock frequency.

Superscalar

Multiple pipelines execute instructions in parallel. Can issue multiple instructions per cycle.

Out-of-Order Execution

Execute instructions as resources become available, not in program order.

Applications

  • Modern CPUs: All modern processors use pipelining
  • Graphics Processing: GPU pipelines for rendering
  • Network Routers: Packet processing pipelines
  • Signal Processing: DSP pipelines for real-time processing
