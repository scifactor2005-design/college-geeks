Data Representation

How computers store and interpret data

Number Systems

Computers use different number systems to represent data. Understanding these is fundamental to computer organization.

Binary (Base-2)

Uses only 0 and 1. This is the native language of computers.

Decimal: 13
Binary: 1101
Calculation: (1×2³) + (1×2²) + (0×2¹) + (1×2⁰) = 8 + 4 + 0 + 1 = 13

Hexadecimal (Base-16)

Uses 0-9 and A-F. Commonly used for memory addresses and color codes.

Decimal: 255
Hexadecimal: FF
Binary: 11111111

Integer Representation

Unsigned Integers

Represent only non-negative numbers. An n-bit unsigned integer can represent values from 0 to 2ⁿ-1.

Bits Range
8-bit 0 to 255
16-bit 0 to 65,535
32-bit 0 to 4,294,967,295

Signed Integers (Two's Complement)

Most common method for representing negative numbers. The MSB (Most Significant Bit) indicates sign.

8-bit signed range: -128 to +127

Positive: 5 = 00000101
Negative: -5 = 11111011
(Invert bits of 5: 11111010, then add 1: 11111011)

Floating-Point Representation (IEEE 754)

Used to represent real numbers with fractional parts.

Single Precision (32-bit):

  • 1 bit: Sign
  • 8 bits: Exponent
  • 23 bits: Mantissa (fraction)

Double Precision (64-bit):

  • 1 bit: Sign
  • 11 bits: Exponent
  • 52 bits: Mantissa
Number: -12.375
Sign: 1 (negative)
Binary: 1100.011
Normalized: 1.100011 × 2³
Exponent: 3 + 127 (bias) = 130 = 10000010
Mantissa: 10001100000000000000000
Result: 1 10000010 10001100000000000000000

Character Encoding

ASCII (American Standard Code for Information Interchange)

7-bit encoding supporting 128 characters (0-127).

'A' = 65 (decimal) = 01000001 (binary)
'a' = 97 (decimal) = 01100001 (binary)
'0' = 48 (decimal) = 00110000 (binary)

Unicode (UTF-8, UTF-16)

Universal character encoding supporting all world languages and symbols.

  • UTF-8: Variable length (1-4 bytes), backward compatible with ASCII
  • UTF-16: Uses 2 or 4 bytes per character

Boolean Algebra

Foundation of digital logic. Basic operations:

Operation Symbol Description
AND ∧ or · Output 1 if both inputs are 1
OR ∨ or + Output 1 if at least one input is 1
NOT ¬ or ' Inverts the input
XOR ⊕ Output 1 if inputs are different

De Morgan's Laws:

NOT (A AND B) = (NOT A) OR (NOT B)
NOT (A OR B) = (NOT A) AND (NOT B)

Applications

  • Image Processing: Pixels represented as RGB values
  • Audio: Sound waves sampled and digitized
  • Compression: Efficient data encoding (ZIP, JPEG)
  • Cryptography: Binary operations for encryption
  • Networking: IP addresses in binary/hex
