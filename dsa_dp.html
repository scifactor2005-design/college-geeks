Dynamic Programming

Optimization through intelligent subproblem solving

What is Dynamic Programming?

Dynamic Programming (DP) is an algorithmic technique for solving optimization problems by breaking them down into simpler subproblems. It stores the results of subproblems to avoid redundant computations.

Key Principles:

  • Optimal Substructure: Optimal solution contains optimal solutions to subproblems
  • Overlapping Subproblems: Same subproblems are solved multiple times

Approaches

1. Memoization (Top-Down)

Start with the original problem and recursively break it down, storing results in a cache.

// Fibonacci with Memoization
int fib(int n, vector<int>& dp) {
    if (n <= 1) return n;
    if (dp[n] != -1) return dp[n];
    return dp[n] = fib(n-1, dp) + fib(n-2, dp);
}

int fibonacci(int n) {
    vector<int> dp(n+1, -1);
    return fib(n, dp);
}

2. Tabulation (Bottom-Up)

Start with the smallest subproblems and build up to the original problem.

// Fibonacci with Tabulation
int fibonacci(int n) {
    if (n <= 1) return n;
    vector<int> dp(n+1);
    dp[0] = 0;
    dp[1] = 1;
    
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i-1] + dp[i-2];
    }
    return dp[n];
}

Classic DP Problems

1. 0/1 Knapsack

Given weights and values of items, maximize value in a knapsack of limited capacity.

int knapsack(int W, vector<int>& wt, vector<int>& val, int n) {
    vector<vector<int>> dp(n+1, vector<int>(W+1, 0));
    
    for (int i = 1; i <= n; i++) {
        for (int w = 1; w <= W; w++) {
            if (wt[i-1] <= w) {
                dp[i][w] = max(val[i-1] + dp[i-1][w-wt[i-1]], 
                              dp[i-1][w]);
            } else {
                dp[i][w] = dp[i-1][w];
            }
        }
    }
    return dp[n][W];
}

2. Longest Common Subsequence (LCS)

int LCS(string s1, string s2) {
    int m = s1.length(), n = s2.length();
    vector<vector<int>> dp(m+1, vector<int>(n+1, 0));
    
    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (s1[i-1] == s2[j-1])
                dp[i][j] = 1 + dp[i-1][j-1];
            else
                dp[i][j] = max(dp[i-1][j], dp[i][j-1]);
        }
    }
    return dp[m][n];
}

3. Coin Change

Find minimum coins needed to make a given amount.

int coinChange(vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, INT_MAX);
    dp[0] = 0;
    
    for (int i = 1; i <= amount; i++) {
        for (int coin : coins) {
            if (i >= coin && dp[i - coin] != INT_MAX) {
                dp[i] = min(dp[i], dp[i - coin] + 1);
            }
        }
    }
    return dp[amount] == INT_MAX ? -1 : dp[amount];
}

When to Use DP?

  • Problem asks for optimization (min/max)
  • Problem can be broken into overlapping subproblems
  • Future decisions depend on earlier decisions
  • Problem has optimal substructure

Common DP Patterns:

  • Fibonacci-like sequences
  • Grid/Matrix path problems
  • Subset sum variations
  • String matching problems
  • Game theory problems

Applications

  • Resource Allocation: Optimizing budget distribution
  • Bioinformatics: DNA sequence alignment
  • Economics: Portfolio optimization
  • Computer Graphics: Image compression
  • Operations Research: Scheduling and planning
