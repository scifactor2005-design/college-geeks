Lexical Analysis

The first phase of compilation - tokenizing source code

What is Lexical Analysis?

Lexical analysis (scanning) is the first phase of a compiler. It reads the source code as a stream of characters and groups them into meaningful sequences called tokens.

Role of Lexical Analyzer:

  • Read source code character by character
  • Group characters into lexemes (token instances)
  • Generate tokens for the parser
  • Remove whitespace and comments
  • Report lexical errors

Tokens

A token is a pair consisting of a token name and an optional attribute value.

Token Categories:

Token Type Examples Pattern
Keywords if, while, int, return Reserved words
Identifiers variable names, function names [a-zA-Z][a-zA-Z0-9]*
Operators +, -, *, /, ==, != Specific symbols
Literals 123, 3.14, "hello" Numbers, strings
Punctuation ;, {, }, (, ) Delimiters 
Source: int sum = a + b;

Regular Expressions

Regular expressions are used to specify patterns for tokens.

Basic Operations:

  • Concatenation: ab (a followed by b)
  • Alternation: a|b (a or b)
  • Kleene Star: a* (zero or more a's)
  • Positive Closure: a+ (one or more a's)
  • Optional: a? (zero or one a)
Identifier: [a-zA-Z][a-zA-Z0-9]*
Integer: [0-9]+
Float: [0-9]+\.[0-9]+
String: \"[^\"]*\"

Finite Automata

Regular expressions are implemented using finite automata.

DFA (Deterministic Finite Automaton)

  • For each state and input, exactly one transition
  • Efficient for pattern matching
  • Used in lexical analyzers

NFA (Non-deterministic Finite Automaton)

  • Multiple possible transitions for same input
  • Easier to construct from regex
  • Converted to DFA for implementation

Lexical Analyzer Implementation

// Simple lexer pseudocode
class Lexer {
    string input;
    int position;
    
    Token getNextToken() {
        skipWhitespace();
        
        if (isLetter(current())) {
            return readIdentifierOrKeyword();
        }
        if (isDigit(current())) {
            return readNumber();
        }
        if (current() == '"') {
            return readString();
        }
        if (isOperator(current())) {
            return readOperator();
        }
        
        return Token(ERROR, current());
    }
    
    Token readIdentifierOrKeyword() {
        string lexeme = "";
        while (isLetterOrDigit(current())) {
            lexeme += current();
            advance();
        }
        if (isKeyword(lexeme))
            return Token(KEYWORD, lexeme);
        return Token(ID, lexeme);
    }
}

Error Handling

Lexical errors occur when input doesn't match any token pattern.

Common Errors:

  • Invalid characters
  • Malformed numbers (e.g., 12.34.56)
  • Unterminated strings
  • Invalid identifiers

Error Recovery:

  • Skip invalid character and continue
  • Report error with line number
  • Try to recover and find next valid token

Applications

  • Compilers: First phase of compilation
  • Text Editors: Syntax highlighting
  • Search Engines: Query parsing
  • Data Validation: Input format checking
