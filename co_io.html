I/O Systems

Connecting computers to the external world

I/O Devices

Input/Output devices allow computers to interact with users and other systems.

Input Devices:

  • Keyboard, Mouse, Touchscreen
  • Scanners, Cameras
  • Microphones
  • Sensors (temperature, pressure, etc.)

Output Devices:

  • Monitors, Printers
  • Speakers
  • Actuators, Motors

Storage Devices:

  • Hard Drives (HDD), Solid State Drives (SSD)
  • USB Drives, SD Cards
  • Optical Drives (CD/DVD)

I/O Communication Methods

1. Programmed I/O (Polling)

CPU continuously checks device status.

Advantages: Simple to implement

Disadvantages: Wastes CPU cycles

2. Interrupt-Driven I/O

Device sends interrupt signal when ready. CPU can do other work while waiting.

Process:

  1. CPU initiates I/O operation
  2. CPU continues other work
  3. Device sends interrupt when done
  4. CPU handles interrupt (saves state, executes ISR)
  5. CPU resumes previous work

3. Direct Memory Access (DMA)

Device transfers data directly to/from memory without CPU intervention.

Benefits:

  • Frees CPU for other tasks
  • Faster data transfer
  • Efficient for large data blocks

DMA Steps:

  1. CPU programs DMA controller (source, destination, size)
  2. DMA controller takes control of bus
  3. Data transferred directly between device and memory
  4. DMA sends interrupt when complete

Bus Architecture

A bus is a communication pathway connecting different components.

Types of Buses:

Bus Type Purpose Example
Data Bus Transfer data 32-bit, 64-bit
Address Bus Specify memory locations 32-bit = 4GB addressable
Control Bus Control signals Read, Write, Clock

Bus Standards:

  • PCI/PCIe: Peripheral Component Interconnect
  • USB: Universal Serial Bus
  • SATA: Serial ATA for storage
  • I2C, SPI: Low-speed serial buses

I/O Performance

Factors Affecting Performance:

  • Bandwidth: Data transfer rate (MB/s, GB/s)
  • Latency: Time to initiate transfer
  • CPU Overhead: CPU time spent on I/O
  • Bus Contention: Multiple devices competing for bus

Optimization Techniques:

  • Buffering: Temporary storage to smooth data flow
  • Caching: Store frequently accessed data
  • Spooling: Queue I/O requests (printing)
  • I/O Scheduling: Optimize order of I/O operations

Device Drivers

Software that allows OS to communicate with hardware devices.

Functions:

  • Initialize device
  • Handle device-specific operations
  • Manage device errors
  • Provide uniform interface to OS

Driver Layers:

  1. Application Layer: User programs
  2. OS I/O Subsystem: Generic I/O services
  3. Device Driver: Device-specific code
  4. Hardware: Physical device

Applications

  • Real-Time Systems: Sensor data acquisition
  • Multimedia: Audio/video streaming
  • Networking: Network interface cards
  • Gaming: Controller input, graphics output
  • Industrial Control: Automation systems
